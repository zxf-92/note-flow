## 1. 技术选型

### 为什么选择这个技术栈？

| 组件 | 技术 | 原因 |
|------|------|------|
| 桌面框架 | Tauri 2.x | 比 Electron 轻量（包小 ~10MB），Rust 后端性能好 |
| 前端框架 | React 18 | 生态成熟，组件丰富 |
| 构建工具 | Vite | 速度快，热更新体验好 |
| 富文本编辑器 | TipTap | 基于 ProseMirror，专业级编辑器 |
| 状态管理 | Zustand | 轻量、简洁，比 Redux 好上手 |
| 本地存储 | IndexedDB (Dexie.js) | 浏览器原生存储，无需后端 |
| 样式 | Tailwind CSS | 原子化 CSS，开发效率高 |

### 对比 Electron

| 指标 | Tauri | Electron |
|------|-------|----------|
| 包大小 | ~10MB | ~150MB |
| 内存占用 | 低 | 高 |
| 生态 | 较小 | 成熟 |
| Rust 后端 | ✅ | ❌ |

## 2. 环境准备

### 2.1 Node.js

推荐使用 Node.js 18+ 或 20 LTS。

**安装方式**：
- Windows: [官网下载](https://nodejs.org/) 或使用 [nvm-windows](https://github.com/coreybutler/nvm-windows)
- 验证: `node --version`

### 2.2 Rust

Tauri 基于 Rust，需要安装 Rust 工具链。

**安装方式**：
```bash
# Windows: 下载 rustup-init.exe
# https://rustup.rs/

# macOS/Linux:
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

**验证**：
```bash
rustc --version
cargo --version
```

### 2.3 Tauri CLI

```bash
npm install -g @tauri-apps/cli
```

### 2.4 Visual Studio Build Tools (Windows)

Tauri 编译需要 C++ 构建工具。

**安装方式**：
1. 下载 [Visual Studio Build Tools](https://visualstudio.microsoft.com/downloads/#build-tools-for-visual-studio)
2. 安装时选择 "C++ Build Tools"
3. 确保安装 MSVC v143 或对应版本

## 4. 前端开发

### 4.1 数据层 (IndexedDB + Dexie)

```ts
// src/db/index.ts
import Dexie, { type Table } from "dexie";

interface Note {
  id: string;
  title: string;
  content: string;
  plainText: string;
  folderId: string;
  tags: string[];
  isFavorite: boolean;
  isAI: boolean;
  createdAt: number;
  updatedAt: number;
  isDeleted: boolean;
  deletedAt: number | null;
}

interface Folder {
  id: string;
  name: string;
  icon: string;
  parentId: string | null;
  order: number;
  isSystem: boolean;
  createdAt: number;
}

export class NoteFlowDB extends Dexie {
  notes!: Table<Note>;
  folders!: Table<Folder>;
  tags!: Table<Tag>;

  constructor() {
    super("NoteflowDB");
    this.version(1).stores({
      notes: "id, title, folderId, isFavorite, isDeleted, updatedAt, createdAt",
      folders: "id, parentId, order, isSystem",
      tags: "id, name, createdAt",
    });
  }
}

export const db = new NoteFlowDB();
```

### 4.2 状态管理 (Zustand)

```ts
// src/stores/appStore.ts
import { create } from "zustand";
import { db } from "@/db";
import { v4 as uuidv4 } from "uuid";

export const useAppStore = create((set, get) => ({
  notes: [],
  folders: [],
  tags: [],
  currentNoteId: null,
  isInitialized: false,

  initialize: async () => {
    // 初始化默认数据
    const folderCount = await db.folders.count();
    if (folderCount === 0) {
      await db.folders.bulkAdd([
        { id: "all-notes", name: "全部笔记", icon: "📁", parentId: null, order: 0, isSystem: true, createdAt: Date.now() },
        { id: "favorites", name: "收藏", icon: "⭐", parentId: null, order: 1, isSystem: true, createdAt: Date.now() },
        { id: "recycle-bin", name: "回收站", icon: "🗑️", parentId: null, order: 2, isSystem: true, createdAt: Date.now() },
      ]);
    }
    const notes = await db.notes.orderBy("updatedAt").reverse().toArray();
    const folders = await db.folders.orderBy("order").toArray();
    const tags = await db.tags.orderBy("createdAt").toArray();
    set({ notes, folders, tags, isInitialized: true });
  },

  createNote: async (folderId = "all-notes") => {
    const note = {
      id: uuidv4(),
      title: "无标题笔记",
      content: "",
      plainText: "",
      folderId,
      tags: [],
      isFavorite: false,
      isAI: false,
      createdAt: Date.now(),
      updatedAt: Date.now(),
      isDeleted: false,
      deletedAt: null,
    };
    await db.notes.add(note);
    const notes = await db.notes.orderBy("updatedAt").reverse().toArray();
    set({ notes, currentNoteId: note.id });
    return note;
  },
}));
```

### 4.3 TipTap 编辑器

```tsx
// src/components/Editor/NoteEditor.tsx
import { useEditor, EditorContent } from "@tiptap/react";
import StarterKit from "@tiptap/starter-kit";
import Underline from "@tiptap/extension-underline";
import Placeholder from "@tiptap/extension-placeholder";

export default function NoteEditor({ note, onUpdate }) {
  const editor = useEditor({
    extensions: [
      StarterKit.configure({ heading: { levels: [1, 2, 3] } }),
      Underline,
      Placeholder.configure({ placeholder: "开始写作..." }),
    ],
    content: note.content ? JSON.parse(note.content) : "",
    onUpdate: ({ editor }) => {
      // 3秒后自动保存
      const json = JSON.stringify(editor.getJSON());
      const text = editor.getText();
      onUpdate(note.id, { content: json, plainText: text });
    },
  });

  return <EditorContent editor={editor} />;
}
```

## 6. 常见问题与解决方案

### Q1: Rust 工具链安装失败？

**问题**: `rustc: command not found` 或工具链不完整

**解决**:
```bash
# 重新安装 stable 工具链
rustup toolchain uninstall stable
rustup toolchain install stable
rustup default stable
```

### Q2: 链接器找不到文件？

**问题**: `ld: cannot find ... .rlib: No such file or directory`

**原因**: Rust 工具链与 MinGW 工具链不兼容

**解决**: 切换到 MSVC 工具链
```bash
rustup toolchain install stable-x86_64-pc-windows-msvc
rustup default stable-x86_64-pc-windows-msvc
```

### Q3: 构建时 exe 文件被占用？

**问题**: `failed to remove file ... note-flow.exe: Access denied`

**解决**:
```bash
# Windows: 结束运行中的进程
taskkill /F /IM "note-flow.exe"

# 或使用任务管理器结束
```

### Q4: 桌面应用一直显示加载中？

**问题**: App 启动后白屏，显示"加载中..."

**排查**:
1. 打开开发者工具查看控制台错误
2. 检查 IndexedDB 是否正常初始化
3. 检查数据库索引是否正确

**解决**: 确保数据库表的索引字段与查询一致
```js
// tags 表必须包含 createdAt 索引
this.version(1).stores({
  tags: "id, name, createdAt",  // 必须包含 createdAt
});
```

### Q5: Tauri devUrl 指向 localhost？

**问题**: Release 构建后提示 "localhost 拒绝连接"

**原因**: Release 构建时 devUrl 不会生效，但配置错误可能导致问题

**解决**: 确保 `tauri.conf.json` 中 `frontendDist` 指向正确的前端构建目录
```json
{
  "build": {
    "frontendDist": "../dist",
    "devUrl": "http://localhost:1420"
  }
}
```

### Q6: npm 安装依赖版本不匹配？

**问题**: `npm error No matching version found for dexie-react-hooks`

**解决**: 使用正确的版本号
```bash
# 查看可用版本
npm view dexie-react-hooks versions

# 使用最新稳定版
npm install dexie-react-hooks@4.0.0
```

### Q7: Rust crates.io 下载慢？

**问题**: `warning: spurious network error: Timeout was reached`

**解决**: 配置国内镜像
```toml
# ~/.cargo/config.toml
[source.crates-io]
replace-with = 'ustc'

[source.ustc]
registry = "sparse+https://mirrors.ustc.edu.cn/crates.io-index/"
```

### Q8: 路径中有空格导致构建失败？

**问题**: `linking with link.exe failed: exit code: 1`

**原因**: 项目路径包含空格（如 `C:\Users\用户名\`）

**解决**: 将项目移到无空格的路径，如 `C:\Projects\note-flow`

### 7.4 常见推送问题

**问题**: `Failed to connect to github.com port 443`

**解决**:
1. 检查网络连接
2. 配置代理（如果有）:
   ```bash
   git config --global http.proxy http://127.0.0.1:端口号
   git config --global https.proxy http://127.0.0.1:端口号
   ```
3. 多试几次，网络波动可能导致临时失败
